# Smart Payment Failure Analyzer 🚀

An AI-powered system that classifies payment failure logs, generates retry suggestions, and converts raw technical errors into plain-English business insights using **Groq LLaMA 3.3 70B** (blazing fast inference).

## Architecture

```
raw logs → rule classifier → LLM classifier (fallback) → suggestion engine → LLM insight → dashboard
```

## Quick Start

### 1. Backend (FastAPI)

```powershell
cd backend
pip install -r requirements.txt

# Copy and fill in your Groq key
copy .env.example .env
# Edit .env and add: GROQ_API_KEY=gsk_...

uvicorn main:app --reload
# → http://localhost:8000
# → http://localhost:8000/docs  (Swagger UI)
```

### 2. Frontend (React + Vite)

```powershell
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

### 3. Docker (all-in-one)

```powershell
# Create .env in root with ANTHROPIC_API_KEY=sk-ant-...
docker-compose up --build
# Frontend → http://localhost:5173
# Backend  → http://localhost:8000
```

## Features

| Feature | Description |
|---|---|
| 🔍 Rule-based classifier | Instant deterministic classification via error code lookup table |
| 🤖 LLM classifier | Groq LLaMA 3.3 70B handles ambiguous / free-text logs |
| 💡 Suggestion engine | Retry timing + alternate payment method for each failure type |
| 🧠 Human insight | 2-sentence plain English explanation generated by Groq LLaMA |
| 📊 Dashboard | Pie chart, trend line, confidence breakdown |
| 📂 Log uploader | Drag-and-drop JSON / CSV or paste directly |

## Failure Categories

| Category | Examples | Suggestion |
|---|---|---|
| `bank_issue` | do_not_honor, card_declined, restricted_card | Retry next business day · suggest UPI/wallet |
| `network_issue` | timeout, connection_reset, service_unavailable | Retry 3× at 30s intervals |
| `insufficient_funds` | nsf, balance_too_low, credit_limit_exceeded | Split payment / BNPL / retry after salary |

## Environment Variables

| Variable | Description |
|---|---|
| `GROQ_API_KEY` | Your Groq API key (gsk_...) — get one free at console.groq.com |

> **Without an API key:** The system works perfectly with rule-based classification only. LLM features degrade gracefully to pre-written fallback messages.
